•
Till now we have been passing explicit values to Oracle queries.
•
We have already seen an example to fetch the record from Supplier table for supplier id = 'S1001'
•
Now, consider a scenario where we want to fetch the details of a particular supplier based on supplier id
but, the value of supplier id should be taken from the user, through Python application at execution time.
•
In this case, we will not be able to pass static values to the query.
•
Here, we can pass the values to query dynamically through bind variables which can be mapped to Python
variables.
•
Two ways of using bind variables:
•
Positional
: execute(query(:bindVariable1,:bindVariable2), (
value1, value2
)
•
Named
: execute(query(:bindVariable1, :bindVariable2), {bindVariable1 ',
value1
, ' bindVariable2,
value2
})
Note:
value1
and
value2
can be Python Variable names as well.
