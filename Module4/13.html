Encapsulation:

In general, encapsulation is the inclusion of one thing within another thing so that the included thing is
not apparent (= visible, evident).

In object-oriented programming, encapsulation is the inclusion within a program object of all the
resources need for the object to function - basically, the methods and the data.

The object publishes its public methods. Other objects adhere to these methods to use the object
without having to be concerned with how the object accomplishes it.

The idea is "don't tell me how you do it; just do it."
