•
Reusability:
In OOP’s programs functions and modules that are written by a user can be reused by other
users without any modiﬁcation.
•
Inheritance:
Through this we can eliminate redundant code and extend the use of existing classes. Multiple
inheritance helps classes inherit members from multiple classes.
•
Data Hiding:
By the way of encapsulation, the programmer can hide the data and methods in a class from
other classes. It helps the programmer to build the secured programs.
•
Message Passing:
The technique of message communication (i.e. calling public methods) between objects
makes the interface with external systems easier.
•
Reduced complexity of a problem:
By the way of abstraction, the given problem can be viewed as a
collection of diﬀerent objects. Each object is responsible for a speciﬁc task. The problem is solved by message
passing between the objects. This technique reduces the complexity of the program design.
•
Easy to Maintain and Upgrade:
OOP makes it easy to maintain and modify existing code as new objects can
be created with small diﬀerences to existing ones.
•
Modiﬁability:
it is easy to make minor changes in the data representation or the methods in an OO program.
Changes inside a class do not aﬀect any other part of a program, since the only public interface that the
external world has to a class is through the use of methods.
