• Stored Procedures can be invoked by any client application and they may or may not return a value.
  • Stored Procedure improves performance as
    • They can minimize the number of calls made to database server.
    • The pre-compiled code just needs to be executed upon each procedure call.
  • Stored Procedures use parameters to receive data from and send result to the calling program. Parameters can be passed in three modes.
    • IN : Default mode for a value passed into procedure. Value cannot be changed.
    • OUT : For a parameter to be returned to calling program. Initially NULL and can be set to any value.
    • IN OUT : Can be passed into procedure and returned to calling environment. Value can be changed.
